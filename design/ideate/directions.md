# API Hub — 布局方向

## Direction A — "Unified Dashboard"
**核心原则**：所有功能平铺在一个页面上，用标签页切分不同板块

```
┌──────────────────────────────────────────────────────┐
│  API Hub                                     ⚙️ 🔄  │
│  Status: ● Running :3456          ↑12 req/min       │
├──────────────────────────────────────────────────────┤
│  [ Providers ] [ Models ] [ Keys ] [ Logs ]  [+ Add] │
├──────────────────────────────────────────────────────┤
│  ┌──────────────────────────────────────────────────┐│
│  │  Provider          Status     Model    Latency   ││
│  │──────────────────────────────────────────────────││
│  │  OpenAI            ● Active   8        32ms    ⚙️ ││
│  │  Anthropic         ● Active   6        45ms    ⚙️ ││
│  │  Ollama (本地)     ● Online   3        2ms     ⚙️ ││
│  │  Google Gemini     ⚪ Disabled -        -       ⚙️ ││
│  └──────────────────────────────────────────────────┘│
└──────────────────────────────────────────────────────┘
```

**优化目标**：
- 一目了然的全貌概览
- 符合用户对"管理后台"的预期模式
- 标签页切换快速，减少导航层级

**牺牲**：
- 单页面信息密度较高
- 日志页面需要独立空间，标签页内可滚动

**适合**：用户需要频繁切换查看不同板块

## Direction C — "Terminal-Inspired"
**核心原则**：开发者风格，强调实时日志和调试

```
┌──────────────────────────────────────────────────────┐
│  API Hub — localhost:3456                   ⏻ ⚙️   │
├──────────────────────────────────────────────────────┤
│  $ models ls                                            │

│  │  PROVIDER    MODEL              STATUS  LATENCY  ││

│  │  openai      gpt-4o-mini       ● ok    28ms     ││
```

**牺牲**：
- 非技术用户可能不适应
- 配置功能需要另外的 UI（弹窗或侧边栏）
- 信息展示效率低于表格

**适合**：开发者/进阶用户，强调调试和实时监控

## 推荐方案

**推荐 A — Unified Dashboard**，原因：
1. 符合竞品（One API、LiteLLM）的用户预期，学习成本低
2. Tab 布局与 DevNexus 现有设计系统一致（容器管理也是 Tab 布局）
3. 信息密度可控，每个 Tab 专注一个功能
4. 扩展性好 —— 未来加功能只需加 Tab

请选择方向 A、B 或 C。
